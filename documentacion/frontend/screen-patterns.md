## 1. Listing Screens

Listings exist for:

- scanning
- filtering
- locating
- comparing

Listings are NOT detailed entity representations.

Listings should prioritize:

- readability
- quick search
- operational filtering
- bulk visibility

Detailed understanding belongs to:

- entity profile screens
- timelines
- contextual views

---

## 2. Entity Profile Screens

Entity screens should feel:

- narrative
- contextual
- operational
- temporal

They should combine:

- current state
- event history
- operational actions
- contextual relationships
- health indicators
- alerts

---

## 3. Dashboard Philosophy

Dashboards are:

- operational summaries
- situational awareness tools
- monitoring surfaces

Dashboards are NOT:

- dumping grounds for widgets
- analytics overload

Each widget must answer:

- what requires attention?
- what changed?
- what blocks operations?
- what needs action?

---

## 4. Calendar Philosophy

The calendar is:

- an operational planning interface
- a temporal coordination tool

NOT merely a visual date grid.

Calendar events should communicate:

- category
- urgency
- operational impact
- dependencies
