## 1. Core Directive

When generating frontend code:

- prioritize semantics over decoration
- prioritize operational meaning over visual novelty
- prioritize traceability over minimalism
- prioritize chronology over static presentation

---

## 2. Forbidden Patterns

Avoid:

- generic admin templates
- random dashboard widgets
- excessive borders
- unnecessary animations
- artificial gamification
- flat CRUD-centric thinking
- purely decorative complexity

---

## 3. Preferred Patterns

Prefer:

- contextual side panels
- event timelines
- operational summaries
- narrative layouts
- grouped workflows
- semantic visual hierarchy
- state explanation

---

## 4. Design Tone

The interface should feel:

- calm
- premium
- trustworthy
- biological
- operational
- editorial
- traceable
- grounded

The system should never feel:

- corporate ERP-like
- chaotic
- hyper-saturated
- playful
- gamified
- startup-generic
