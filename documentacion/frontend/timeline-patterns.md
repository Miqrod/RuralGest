## 1. Timeline Philosophy

Timelines are first-class UI structures.

The system should naturally evolve toward:

- event streams
- operational narratives
- chronological understanding

not static CRUD tables.

---

## 2. Timeline Requirements

A good timeline must show:

- timestamp
- event type
- actor/source
- affected entities
- operational consequence
- resulting state if relevant

---

## 3. Timeline Visual Hierarchy

Primary:

- critical events
- biological milestones
- irreversible operations

Secondary:

- informational updates
- automated recalculations
- low-impact changes

---

## 4. Temporal Navigation

Users should be able to:

- scan history quickly
- identify anomalies
- understand causality
- reconstruct operations

The timeline should support investigation.
