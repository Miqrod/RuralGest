## 1. Responsive Philosophy

Responsive design must preserve:

- operational clarity
- hierarchy
- traceability
- scanning efficiency

Responsive does NOT mean:

- collapsing everything into cards blindly
- hiding important data
- removing operational context

---

## 2. Mobile Priorities

Mobile layouts should prioritize:

1. alerts
2. active workflows
3. operational actions
4. current state
5. contextual history

Avoid:

- giant forms on mobile
- dense tables without restructuring
- tiny tap targets

---

## 3. Desktop Philosophy

Desktop should leverage:

- parallel information
- side panels
- contextual timelines
- operational density

Desktop should feel like:

- a professional operations console

not a stretched mobile app.
